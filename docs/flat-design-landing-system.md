## 2. 设计目标

主站首页的角色不是产品详情页，而是统一入口页。

这次改版的目标有四个：

1. 让首页更像一个完整的数字海报，而不是若干漂浮模块拼接
2. 用统一的 Flat Design 语言承接整个 `ijaa` 站群
3. 明确说明主站与两个子站的关系
4. 在最短阅读路径内把用户分发到对应产品

首页需要解决的问题是：

- `ijaa.github.io` 到底是什么
- 下面有哪些产品
- 三者的关系是什么
- 应该点哪里继续进入

## 3. 设计哲学

本次首页采用明确的 Flat Design，而不是玻璃拟态或通用 SaaS 浮层风格。

核心原则如下：

### 3.1 Zero Artificial Depth

不使用：

- box-shadow
- backdrop blur
- 漂浮发光
- 拟物式层次

视觉层级完全由：

- 色块
- 字号
- 字重
- 排版位置

来建立。

### 3.2 Color As Structure

背景颜色不是装饰，而是页面结构本身。

首页用：

- 蓝色首屏
- 白色产品区
- 灰色系统说明区
- 深色收尾区

来划分阅读阶段。

### 3.3 Typography As Hierarchy

首页不靠复杂装饰吸引注意，而靠标题系统：

- 大标题负责第一层级
- eyebrow 负责定位
- lead 负责解释
- 卡片标题负责分发决策

### 3.4 Poster Composition

首屏的目标不是“像网页”，而是“像一张能点击的海报”。

因此采用：

- 整块单色背景
- 几何图形组合
- 下方嵌入的纯白信息卡
- 强烈但克制的配色对比

## 4. 统一视觉系统

### 4.1 颜色 Token

主站首页使用如下 token：

- `Background`: `#FFFFFF`
- `Foreground`: `#111827`
- `Primary`: `#3B82F6`
- `Primary Strong`: `#2563EB`
- `Secondary`: `#10B981`
- `Accent`: `#F59E0B`
- `Muted`: `#F3F4F6`
- `Muted Strong`: `#E5E7EB`
- `Dark`: `#111827`
- `White`: `#FFFFFF`

用法规则：

- `Primary` 用于首屏主背景和核心品牌感
- `Secondary` 用于产品体系中的辅助强调
- `Accent` 用于数值亮点和视觉跳点
- `Muted` 用于说明区和浅底卡片
- `Dark` 用于收尾区和高对比段落

### 4.2 字体系统

统一字体：

- `Outfit`

原因：

- 几何感强
- 数字产品气质明确
- 用在大标题和按钮上都稳定

排版规则：

- 标题使用 `700 / 800`
- 标题字距统一收紧到 `-0.02em`
- 标签与 eyebrow 使用大写和宽字距
- 正文行高维持在 `1.65 - 1.7`

### 4.3 圆角与边界

统一规则：

- 小圆角：`6px`
- 大圆角：`8px`

避免：

- 大面积胶囊化
- 过于柔软的曲线轮廓

### 4.4 按钮规则

按钮分为两种：

- 实底按钮
- 粗边框 Outline 按钮

交互规则：

- hover 只使用颜色切换和 `scale`
- 不使用阴影强调

### 4.5 卡片规则

卡片不承担“浮起”功能。

卡片只承担：

- 信息承接
- 产品分发
- 数据说明

因此卡片使用：

- 白底
- 浅色底
- 无阴影
- 中等 padding

## 5. 首页信息结构

首页正式版采用四段结构。

## 5.1 首屏 Hero

首屏是一个完整的单色块海报区。

组成：

- 顶部导航
- 左侧价值主张
- 右侧几何海报构图
- 底部嵌入的三张信息卡

设计要求：

- 导航和 hero 不能拆成两个背景色块
- 首屏必须作为一个整体被感知
- 用户进入页面后不需要滚动就能理解站点角色

首页 Hero 传达的信息是：

- 这是 `ijaa` 的统一入口
- 当前站群已经统一到新的 Landing 语言
- 可以直接进入两个产品

## 5.2 Products 区

第二段是产品分发区。

功能：

- 解释两个产品各自是什么
- 解释主站在整个体系中的作用
- 给出直接跳转入口

这一区域包含四张卡：

1. `Baby Future`
2. `Image Story`
3. `GitHub Pages 聚合发布`
4. `Shared Flat Tokens`

这四张卡的设计逻辑不是展示功能细节，而是让用户快速理解：

- 有哪两个产品
- 为什么会在同一域名下
- 这不是随意拼接，而是一套统一系统

## 5.3 System 区

第三段是系统说明区。

功能：

- 解释本次首页改版到底改变了什么
- 用较短路径说明视觉系统的统一规则

该区用三张指标卡表达：

- Typography
- Color Blocking
- Feedback

它们的作用是把视觉抽象为明确规则，而不是停留在“好看”层面。

## 5.4 Routing 区

最后一段是收尾区。

功能：

- 再次确认当前页是正式 Landing 体系的一部分
- 强化主站作为分发器和系统入口的角色

这一段不承担复杂转化，只承担收口。

## 6. 响应式策略

首页没有采用固定高度模块来“撑满屏”，而是采用视口预算策略。

核心方法：

- 首屏高度跟随视口
- 标题使用 `clamp(...)`
- 海报区高度跟随 `svh`
- 在矮屏场景收紧垂直间距

这样处理的原因：

- 固定高度在不同设备下容易溢出
- 首页首屏必须保证整体完整
- 视觉节奏应该收缩，而不是强迫用户一进入就滚动

## 7. 技术实现

主站仍然使用单文件静态 HTML：

- [site/index.html](/Users/liukai/Documents/workspace/other/kkstudios/ijaa.github.io/site/index.html)

原因：

- 首页交互复杂度低
- 不需要引入额外前端框架
- 发布路径最短
- 对 GitHub Pages 最友好

实现特点：

- 样式内联在页面中
- 通过 `@import` 使用 `Outfit`
- 不依赖构建时组件抽取
- 以结构清晰、可直接发布为优先

## 9. 维护约束

后续如果继续修改首页，必须遵守以下约束：

1. 不要重新引入阴影系统
2. 不要把首屏重新拆成两个背景区
3. 不要把主站做成产品详情页
4. 不要在首页塞入过多解释性内容，打乱分发效率
5. 不要把系统说明区做成空洞的设计自夸，必须服务于理解产品关系

## 10. 后续可演进方向

主站后续可继续优化的方向：

- 进一步打磨移动端首屏海报构图
- 补充更明确的站群关系说明
- 给产品卡增加更强的状态信息，例如版本、入口类型、是否公开测试
- 把首页系统说明进一步压缩，使分发更直接

## 11. 结论

主站首页这次改版的本质不是换皮，而是重新定义它作为“统一入口页”的视觉角色。

新版首页完成了三件事：

- 建立统一的 Flat Design 识别
- 把两个产品放进同一套系统叙事里
- 用更短路径把用户分发到目标页面

这份文档即为当前主站正式 Landing 的设计基准。
